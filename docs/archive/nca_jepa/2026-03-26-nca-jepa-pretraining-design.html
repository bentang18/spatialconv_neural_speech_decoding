Synthetic Grid-Dynamics Pretraining for Motor-Speech Decoding from Acute Micro-ECoG — Design Spec

Date: 2026-03-26 (v14) Author: Ben Tang, Cogan Lab (Duke) Status: Draft — design-review version, ready to convert into implementation work after meeting discussion Changelog: nca-jepa-changelog.md | Resolved decisions: nca-jepa-resolved-decisions.md (82 entries)

1. Problem Statement

Intraoperative uECOG speech decoding faces an extreme data bottleneck: ~1 minute of utterance per patient (~150 trials), 128-256 channels at 200Hz. Current supervised approaches train ~60K parameters from scratch on this thin data and overfit heavily — a ~330K GRU backbone showed a 55x train/val gap when trained from scratch. SSL methods exist for speech-BCI (wav2vec ECoG, BrainBERT), but none target the specific regime of <200 trials on rigid micro-ECoG arrays with known 2D spatial structure. Transformers are standard for SSL but typically require orders of magnitude more data than GRUs (BIT needed 367 hrs); this spec tests whether synthetic pretraining can pay for that capacity gap.

2. Core Idea

Pretrain an encoder-predictor on synthetic spatiotemporal grid dynamics, then adapt to real cortical recordings, then fine-tune a lightweight classification head per patient.

The synthetic data generators are organized as a comparative ladder of inductive biases (Section 4.1): from generic spatial smoothness (Gaussian AR) through regime-switched linear fields and local coupling (switching LDS, wave PDE) to nonlinear local-rule systems (FHN, NCA). Each level adds a primary new property, though secondary characteristics also change (see confound acknowledgment in Section 4.1). If Level N > Level N-1, we can say the added property is associated with the improvement, not that it caused it. Each generator’s output statistics are explicitly compared to real uECOG data to ground the transfer hypothesis. A matched-statistics destroyed-dynamics control (Method K) tests whether the benefit comes from structured temporal dynamics or merely from exposure to similar spatial statistics.

Three architecture variants are tested: (A) a smaller conv-GRU (~71K params) that deliberately preserves the existing supervised pipeline’s spatial-collapse architecture as a conservative control; (A.5) a conv-token GRU bridge (~40K shared params) that preserves spatial tokens while keeping the GRU temporal model, isolating the spatial-preservation hypothesis; and (B) a ViT + factorized transformer (~930K params) that preserves spatial tokens through a factorized space-time self-attention predictor but requires freezing. Architecture is an experimental axis, not a default.

The synthetic-neural analogy: Both synthetic generators and uECOG produce 2D grids evolving over time. The model must extract spatial patterns and track their temporal evolution. The analogy is imperfect: synthetic rules are deterministic while cortical dynamics are stochastic; most synthetic rules are fixed per sequence while neural trials have within-trial regime switches across 3 phonemes; synthetic grids are homogeneous while real arrays have dead electrodes, patient-specific anatomy, and spatially correlated noise. Stage 1 data bridges these gaps via nuisance augmentation and regime-switching dynamics (see Section 4).

Pretraining objective: Masked temporal span prediction in observation space. Mask 40-60% of frames using 3-6 non-overlapping spans and predict the missing frames from bidirectional context. This objective is compatible with both architecture families without modification (BiGRU is inherently bidirectional; ViT uses bidirectional attention). The exact pretext task (masked spans vs causal multi-step prediction) is an ablation axis.

Synthetic JEPA / latent target prediction as the first promoted follow-up: We start with observation-space prediction for simplicity and stability, but latent prediction is the first follow-up branch if raw reconstruction looks shortcut-prone. HGA contains substantial patient-specific nuisance variation (impedance differences, dead channels, local smoothness) where JEPA-style abstraction could be materially better than pixel/voxel-space fidelity. Promotion criteria (any one is sufficient): (a) event-frame pretext gains do not survive execution-only decoding, (b) patient-ID probe rises >2σ above random without phoneme-ID improvement, or (c) pretext MSE plateaus for >5K steps while transfer-proxy accuracy stagnates. If triggered, the next follow-up run is synthetic JEPA / latent target prediction before broadening the generator matrix further.

3. Paper Scope

Title: Synthetic Grid-Dynamics Pretraining for Motor-Speech Decoding from Acute Micro-ECoG

Claims (three questions, scoped to motor-speech decoding): 1. Does SSL pretraining help for acute intra-op uECOG motor-speech decoding at <200 trials/patient? 2. Does synthetic pre-pretraining add value beyond real-data-only SSL? 3. Does structured synthetic dynamics help beyond matched-statistics non-dynamic exposure?

Primary endpoint: Full-window readout [-0.5, +1.0s] — the validated configuration (PER 0.700 baseline used full-window pooling). This is the headline result for all main comparisons.

Preregistered mechanistic secondary: Motor-focused readout [-0.15, +0.6s] (Phases C-E, excluding auditory). Always reported alongside full-window. If motor-focused > full-window, auditory was nuisance; if full-window > motor-focused, the crop discards useful context. The divergence is informative, but the headline uses full-window to avoid choosing the friendlier story post-hoc.

Validation: uECOG dataset (8-12 patients). Second dataset deferred.

Relationship to existing codebase: Separate pipeline. Shares data loading utilities but has its own models, training, and evaluation.

4. Three-Stage Pipeline

Stage 1: Synthetic Pre-pretraining (unlimited synthetic data)

Goal: Teach encoder and predictor to process 2D grids evolving over time.

Synthetic data generator ladder — ordered by cumulative inductive bias, each level adding a primary new property (though secondary characteristics also change — see confound note below). All generators produce continuous scalar values per cell:

Level Generator Local coupling Nonlinear Diverse rules New property added
0 Smooth Gaussian AR — — — (baseline: spatially smooth, temporally correlated)
1 Switching Linear State-Space Field Yes — — + Regime switching / piecewise-linear dynamics
2 Damped Wave Yes — — + Oscillatory local coupling (linear PDE)
3 Gray-Scott Yes Yes — + Nonlinearity (reaction-diffusion)
4 FitzHugh-Nagumo Yes Yes — + Excitable-medium dynamics (qualitative toy model of propagating activity, not a model of HGA per se)
5 NCA Random MLP Yes Yes Yes + Rule diversity (different random MLP per sequence produces varied dynamics)

Generator details:

  1. Spatially-Smoothed Gaussian AR (control) - AR(1) process: x_{t+1} = alpha * x_t + (1-alpha) * smooth(noise), sigma=2-4 cells - No local update rules, no emergent dynamics - Tests whether ANY smooth synthetic movie helps, regardless of structure

  2. Switching Linear State-Space Field - Piecewise-linear dynamics: x_{t+1} = K_{s_t} * x_t + b_{s_t} + eps_t, where s_t is a regime index and K_{s_t} is a stable local 3x3 convolutional kernel - 3-6 regimes per sequence with dwell times matched to the synthetic scaffold phases and sub-events - Tests whether regime switching plus stable linear local dynamics is already enough before invoking PDE/nonlinear generators

  3. Damped Wave Propagation - Wave equation with random speed, damping, initial impulses - Traveling waves with local coupling but linear dynamics

  4. Reaction-Diffusion (Gray-Scott) - Two variables per cell (u activator, v inhibitor), model observes u only - Parameters (F, k) sampled per sequence from known interesting regime - Turing patterns, waves, spots

  5. Excitable Media (FitzHugh-Nagumo) - Two variables per cell (v membrane potential, w recovery), model observes v only - Parameters (a, b, epsilon, D) sampled per sequence - Note: this is a qualitative toy model — excitable media produce propagating activation fronts on 2D grids, which is the geometric property we want, not a claim about cortical biophysics

  6. NCA Random MLP (from Lee et al. 2026) - Rule: MLP(9 -> 32 -> 1), tanh hidden, random weights per sequence - 3x3 neighborhood, synchronous update - Produces diverse dynamics (chaos, waves, fixed points, oscillations) because rule varies per sequence

Additionally: Sequential Hotspot Activation — Gaussian bump following a random smooth trajectory. Tests trajectory structure vs local-coupling structure.

Key comparisons from the ladder: - Level 0 vs 1: does regime switching / piecewise-linear dynamics help beyond smooth AR? - Level 1 vs 2: does oscillatory local coupling help beyond switching linear fields? - Level 2 vs 3-4: does nonlinearity help? - Level 4 vs 5: does rule diversity help? - Level 0 vs Mixed (all levels): does combining everything help? - If Mixed wins, the paper becomes “diverse synthetic dynamics help” — not “cortex-like dynamics help.” Both interpretations are prepared.

Confound acknowledgment: Each level changes a primary property but also changes secondary characteristics. For example, the switching LDS adds regime structure and changes event persistence relative to smooth AR; Gray-Scott adds nonlinearity and changes spectral shape, stationarity, and event morphology relative to the damped wave. If Level 3 > Level 2, we can say “nonlinearity is associated with improvement” but not strictly “nonlinearity caused it” — Gray-Scott also changes spectral shape and stationarity relative to the damped wave. For causal isolation, pairwise matched controls would be needed, which is deferred. Method K (destroyed-dynamics control) provides a different cut: it tests whether temporal dynamics matter at all, by preserving per-frame statistics while destroying temporal structure. The ladder is designed for efficient comparison across a wide range of inductive biases, not for single-variable causal claims.

Real-data matching statistics (computed for each generator and compared to real uECOG): Before running pretraining, measure the following for both synthetic and real trial data. All real-data statistics are computed on source patients only (excluding the dev patient and any target patient in the current evaluation fold). Generator selection/tuning that uses target patient statistics would leak evaluation information into the training pipeline.

Coarse source-only generator calibration (required before pretraining): We do NOT tune generators to match real statistics closely (that would confound the inductive-bias comparison). However, “no tuning” is too extreme — each generator family has hyperparameters where poor default instantiation produces degenerate dynamics (blowup, collapse to fixed point, wrong timescale) that would be mistaken for a bad inductive bias rather than a bad hyperparameter. The right compromise: a coarse calibration pass, run on source-patient statistics only, that adjusts 1-2 parameters per generator family to land inside broad acceptable bands: - Temporal PSD: main spectral power below 20Hz (not concentrated at DC or aliased) - Spatial covariance length: 2-10 electrode spacings (not white noise, not globally correlated) - Amplitude range: kurtosis > 2 (not Gaussian/uniform, some events) - Event sparsity: 5-30% of cells active at any frame (not all-on or all-off)

This is analogous to checking that a simulation doesn’t blow up — it does not optimize for real-data match. Document the calibrated parameter ranges; report them alongside downstream results.

Nesting rule for calibration: For the primary inductive result, “source patients” means the current fold’s non-dev, non-target patients, so coarse calibration is rerun inside each target-patient fold using only that fold’s source pool. For the secondary transductive result, reuse the same synthetic calibration as the corresponding inductive fold; target unlabeled data may enter Stage 2 neural adaptation, but it does not retune the synthetic prior.

  • Calibration rule: All matching statistics are computed on raw, unclamped HGA. Visualization-only clamping (e.g. setting [0, 2 SD] to zero for hotspot inspection) is useful for qualitative review but must NOT set nuisance scale.
  • Temporal PSD: Welch periodogram of per-cell time series, averaged across cells. Concatenate across trials per patient (zero-mean each trial first) before computing Welch (256-point FFT, Hanning window, 50% overlap). Report median PSD ± IQR across patients. Also compute residual PSD (trial minus trial-average) to separate evoked from dynamic spectral content
  • Spatial covariance length scale: Fit exponential decay to pairwise Pearson correlation vs physical distance (mm). Report e-folding distance. Compute on both trial-averaged (evoked) and single-trial residual data. If evoked length scale is long but residual is short, generators matched to evoked statistics may poorly capture trial-by-trial dynamics
  • Amplitude distribution: histogram of per-cell values across all frames. Report kurtosis and 95th percentile
  • Dead-mask prevalence: fraction of zero/inactive cells (real: structured corners in 12×22, 0-1 random in 8×16)
  • Event sparsity: fraction of frames × cells with |z| > 1.5, computed separately for each trial phase
  • Motor-trial phase structure: Trial-averaged spatial variance (var across electrodes per frame) as a time series. Report peak/baseline spatial variance ratio per patient
  • Cue-to-response delay distribution: RT from stimulus to response onset. Report mean, std, and fitted log-normal parameters per patient. Synthetic trials sample delay from pooled source-patient distribution

Real trial anatomy — 5 phases: Real trial windows are 1.5s response-locked ([-0.5, 1.0s]). The trial is NOT a homogeneous dynamical movie:

|--A: auditory--|--B: delay/plan--|--C: premotor--|--D: execution (3 phon)--|--E: decay/feedback--|
t=-0.5s         t≈-0.3           t≈-0.15         t≈0                       t≈0.45              t=1.0s

Timing is approximate. PhonemeSequence has no fixed planning delay — patients hear and repeat — so cue-to-response RT variability (~300-800ms) is the dominant source of trial-to-trial timing variation.

Phase properties (from real uECOG observations): - A — Auditory/sensorimotor (~200ms): Residual auditory-evoked activity. Broad spatial footprint, not token-specific. Easiest nuisance signal. - B — Delay/planning (~150ms): Sensorimotor transformation. Lower-amplitude, transitional. - C — Premotor buildup (~150ms): Increasingly focal, amplitude rising. Likely useful for decoding. - D — Execution (~450ms): Three phoneme sub-events (~80-200ms each from MFA) with coarticulatory transitions. Focal, highest amplitude, most discriminative. - E — Decay/feedback (~550ms): Post-speech decay, possible auditory feedback, return to baseline.

Motor-focused crop: The encoder/predictor processes the full [-0.5, +1.0s] trial for context, but the headline readout window is full-window (see Stage 3). The motor-focused crop [-0.15, +0.6s] excludes Phase A (auditory nuisance) and is reported as a preregistered mechanistic secondary.

Synthetic trial structure — motor-focused, externally driven: Generators are forced, not free-running:

x_{t+1} = f_theta(x_t) + u_t

where u_t is a phase-dependent external drive signal. The synthetic trial has 3 phases (matching the motor-focused real window):

  • Planning phase (~200-350ms): Low-amplitude focal ramp (small spatial support, ~10-20% of grid), amplitude linearly increasing.
  • Execution phase (~300-500ms): This is where the generator ladder belongs. The switching-LDS / wave / RD / FHN / NCA generator produces the autonomous component f_theta(x_t); external drive u_t provides generic sub-event templates — random focal spatial patterns sampled independently per sub-regime and per sequence, NOT tied to real phoneme identity. Three sub-regimes (~80-200ms each with ~100ms crossfade).
  • Decay phase (~300-500ms): Exponential return toward zero, ~200ms active then silence.

Micro-event structure within execution (ablation, requires 40Hz): The default has 3 coarse sub-regimes. Real motor cortex likely has finer structure. The micro-event ablation adds 2-4 activation motifs per sub-regime (6-12 total), with jittered onset (~50-100ms spacing) and partial overlap. Requires 40Hz: at 20Hz, 50ms motifs occupy ~1 frame and are aliased away.

Timing variability: Phase boundaries have ±100ms jitter (planning-to-execution), phoneme sub-boundaries from MFA timing prior, execution-to-decay ±75ms. Total 800-1350ms.

Scaffold variants (first-round control — Method F): - (a) Random phase count (2-5 sub-regimes at random timepoints) — required to distinguish 3-event curriculum from generic multi-event dynamics - (b) Execution phase only (no planning/decay wrapper) - (c) Full 5-phase scaffold including simulated auditory drive

Nuisance realism (applied to ALL synthetic data): Visual inspection of clamped renders is useful for hotspot sparsity and focality, but it understates raw background variability because low-amplitude activity is zeroed in the render. Nuisance calibration therefore targets raw residual trial statistics, not visualization movies. - Z-score normalize to zero mean, unit variance - IID Gaussian noise: sample per-sequence sigma_iid ~ Uniform(0.3, 0.8) with 0.5 as the default center. This should blur weak events without swamping the ~4-10 SD focal activations seen in real trials - Anti-alias LPF before subsampling - Dead electrode simulation: Real mask templates from known arrays (12×22 structured corners, 8×16 sporadic) plus 0-2 additional random dead positions - Flips and 180-degree rotations (90° deferred — swaps H/W) - Spatially correlated noise: low-rank common-mode fluctuation (rank 1-3) with total SD sampled from Uniform(0.2, 0.6) and 0.4 as the default center - No post-noise re-zscoring: explicit nuisance amplitude is preserved after noise injection so the synthetic SNR remains a real design choice - Phase timing variability (see above) - Grid sizes: 8x16, 12x22, 8x32, 8x34 (proportional to real distribution)

Training objective — masked temporal span prediction in observation space: 

Mask 40-60% of frames using 3-6 non-overlapping spans (each 2-5 contiguous frames).
Predict masked frames from bidirectional context:

  frames (with masked spans) -> Encoder -> spatial tokens -> Predictor -> predicted tokens
  -> Decoder -> predicted patches for masked frames

  Loss = MSE(predicted_patches_masked, actual_patches_masked)

Why 40-60% masking: At 20Hz with 30 frames per trial, the previous 3-5 frame mask (10-17%) was trivially solvable by smooth interpolation — the signal is low-pass and temporally autocorrelated. VideoMAE demonstrated that SSL on smooth signals requires aggressive masking to prevent trivial reconstruction. At 40-60%, the model must reconstruct 12-18 frames from 12-18 observed frames, making linear interpolation between sparse boundaries inadequate. Mask-ratio sweep (30%, 50%, 70%) is a priority ablation (Section 8).

Pretext baselines (required — listed from weakest to strongest):

  1. Interpolation baseline: Linear interpolation between boundary frames flanking each masked span. The trained model must substantially beat this. Necessary but NOT sufficient.

  2. Phase-template baseline: Average frame at each relative trial timepoint. Tests whether the model has learned anything beyond “what does a typical trial look like at this timepoint?”

  3. Patient-specific response-locked template baseline: Trial-averaged activation per timepoint, per patient. Tests whether gains are patient-specific trial scaffolding.

  4. Cropped-window comparison (always run): Evaluate on full trial [-0.5, 1.0s], motor-focused [-0.15, +0.6s], and execution-only [0, 0.5s]. If the gain disappears at execution-only, the model learned phase structure, not dynamics.

  5. Destroyed-dynamics baseline (required alongside Method K): Same synthetic data as Method A (same generators, same scaffold, same spatial statistics) but with temporal dynamics destroyed: - Phase-randomized: FFT each cell’s time series per phase, randomize phase angles, IFFT. Preserves PSD and amplitude distribution; destroys temporal ordering and cross-cell temporal correlations. - Time-shuffled within scaffold: Within each phase, randomly permute frame order. Preserves per-phase spatial statistics; destroys temporal continuity. If A ≈ K (destroyed-dynamics), the benefit is from spatial-statistics exposure, not structured dynamics. If A > K, temporal dynamics are load-bearing for Claim 3.

The trained model must beat all baselines, not just interpolation. Report all metrics in a single table per experiment.

Event-frame diagnostic (required): Compute pretext MSE separately on event frames (cells with |activation| > 1σ) vs background frames. If the model beats interpolation on background but not events, it has learned smooth interpolation. If event MSE plateaus, add event-weighted loss (upweight 3-5×).

Stage 2: Response-Locked Trial Adaptation (fold-specific patient pools, no phoneme labels)

Goal: Adapt encoder and predictor from synthetic to real cortical dynamics using response-locked trial epochs.

Data: Unlabeled response-locked trial epochs from the PhonemeSequence cohort (~1500+ trials total across all usable patients). No phoneme labels used. Lexical task patients are a separate ablation.

Stage 2 pools are fold-specific. Let d be the single fixed dev patient and t the held-out target patient for the current headline fold:

Mode Synthetic calibration pool Stage 2 neural pool Headline evaluation
Primary inductive all PhonemeSequence patients except {d, t} all unlabeled PhonemeSequence trials except {d, t} patient t only
Secondary transductive same as corresponding inductive fold all unlabeled PhonemeSequence trials except {d} (includes target t unlabeled trials) patient t only

The dev patient d is never a headline target. Methods A/B/C/F/K and their Stage 2 neural adaptation are rerun per target-patient fold under these pools.

Supervision level — honest accounting: Stage 2 uses no phoneme labels but is NOT unsupervised. It requires: - Response-onset timestamps (from MFA): needed to epoch the data and define trial windows - Trial identity: every training sample is a speech trial — the model never sees inter-trial silence, resting state, or non-speech activity - Patient identity: needed for per-patient adapters

This makes Stage 2 weakly-supervised trial-epoch adaptation, not robust dynamics SSL. The response-locked trial structure gives the model a phase-calendar shortcut: auditory → planning → execution → decay appears in every sample. This is the same failure mode as the acoustic-regression pivot, where 500ms-shifted controls were competitive because the model learned trial structure, not alignment-sensitive mappings. The mitigations below attack this shortcut directly, but the claim scope must reflect the supervision level.

Full-epoch off-center random crops (primary anti-leakage defense): Load the full [-1.0, 1.5s] epoch (2.5s = 50 frames at 20Hz) and crop to a random contiguous window of 30-40 frames (1.5-2.0s) with random start offset. This genuinely disrupts the phase calendar: - A crop starting at -1.0s sees [-1.0, +0.5s]: heavy on baseline + auditory, cuts off late execution - A crop starting at -0.5s sees [-0.5, +1.0s]: the standard training window - A crop starting at 0.0s sees [0.0, +1.5s]: execution + decay only, no auditory precursor

Unlike the previous 70-100% jitter (which shifted frame indices but preserved the full phase calendar), off-center crops from the wider epoch remove entire phases from some training samples.

Time-reversal augmentation (10% probability): Reverse the temporal order of a cropped window. This destroys causal phase ordering. Combined with bidirectional models, the model should learn dynamics useful in both directions. If time-reversed samples consistently hurt, the model is relying on causal phase structure — an informative diagnostic.

Full-trial evaluation: Stage 2 evaluation uses the full [-0.5, 1.0s] window (no crop) for consistency with Stage 3. The crop augmentation is training-time only.

Phase-rebalanced prediction loss (phase-aware variant, NOT default): Weight reconstruction loss by excess error over the phase-template baseline: sqrt(max(MSE_phase - template_MSE_phase, epsilon)) / sum(...). Upweights phases where the model underperforms the trivial baseline, not phases that are merely high-amplitude. Requires response-locked phase boundary assignments — the phase-aware variant. The pure-SSL default uses uniform loss. Their comparison is an experimental axis.

Training: - Initialize from Stage 1 weights - Per-patient patch embedding (ViT) or per-patient Conv2d (conv-GRU). Decoder shared - Masked temporal span prediction (same objective as Stage 1, same 40-60% masking) - Monitor per-patient prediction loss variance and dominance ratios

Negative transfer mitigation: Uniform patient sampling is the default. Ablations: similarity-weighted (leave-one-out within source patients, NOT dev patient) and loss-balanced sampling.

Transfer-proxy model selection gate: Hold out 1 fixed dev patient (e.g., S26, median trial count/quality). Select before any experiments. - Probe every 5K steps (not 1K — reduces meta-optimization pressure) with a quick linear CE probe on dev patient’s labeled data, using the locked dev split (same fold assignment as all methods) - Earliest-within-1SE rule: select the earliest checkpoint whose transfer-proxy accuracy is within 1 standard error of the best. Penalizes late checkpoints that may overfit to pretext statistics - Source-only sensitivity analysis (required): also track best-pretext-loss-on-source-patients as an alternative checkpoint selector. If source-only and transfer-proxy select similar checkpoints, the dev patient is not load-bearing; if they diverge, report both - Dev patient excluded from all headline claims. Performance reported separately

Required diagnostics: - Per-phase prediction loss (both variants): Track MSE per trial phase. If Phase A drops much faster than Phase D, the model is learning trial structure - Patient-ID and phoneme-ID linear probes: Sharp patient-ID rise without phoneme-ID improvement signals nuisance learning - Content-collapse check: Compute output entropy of the CE probe’s predictions on the dev patient. If predicted phoneme distribution concentrates on 1-3 classes (entropy < 1.5 bits, max = log2(9) ≈ 3.17) or matches the training marginal (high unigram-KL from uniform), encoder features lack discriminative diversity. Track across checkpoints alongside transfer-proxy accuracy

Stage 3: Per-Patient Fine-tuning (labeled data)

Goal: Train a lightweight classification head on pretrained features.

Default freeze configuration: Freeze all weights except classification head, per-patient patch embedding (or per-patient Conv2d), and predictor LayerNorm parameters. ~5.3K trainable for ViT, ~2.3K for conv-GRU, ~2.2K for conv-token GRU.

Training: For each patient independently, grouped-by-token cross-validation. All repetitions of the same CVC/VCV token are assigned to the same fold, preventing token identity leakage. Fold count is adaptive: min(5, n_unique_tokens_with_sufficient_reps). Per-position class coverage constraint: each fold’s training set must contain all 9 phonemes × 3 positions. Fallback: repeated random grouped holdout (80/20 by token group, 10 reps).

Fixed split reuse (required): Generate token-group fold assignments once per patient (deterministic seed from patient ID), save to JSON, reuse across ALL methods. Method variance must not be polluted by split variance.

Comparability note: The 0.700 PER baseline used first-phoneme stratified CV, not grouped-by-token. The 0.700 is cited only as CE > CTC evidence — it is NOT directly comparable to grouped-by-token results.

Decoding window: - Full-window (headline primary): Mean-pool over all frames [-0.5, 1.0s]. Validated configuration (PER 0.700). All main results use this. - Motor-focused (preregistered secondary): Mean-pool over [-0.15, +0.6s] (Phases C-E). Always reported alongside full-window. - Execution-only (ablation): Mean-pool over [0, +0.5s]. Strictest test. - Implementation: Encoder/predictor process all frames. Temporal features are cropped to the readout window before pooling.

Loss options (CE primary): - CE with temporal mean-pool (primary): 27-way head (3 positions × 9 phonemes) applied to mean-pooled backbone features over the full-window. Matches the configuration that produced PER 0.700. - CE with three-bin temporal pooling (secondary probe, always alongside mean-pool): Divide the readout window into 3 equal-duration temporal bins. Each bin mean-pooled independently → 3 separate 9-way CE heads. Zero extra parameters. Tests temporal discriminability: if three-bin > mean-pool, features are temporally structured; if equal, features are temporally diffuse. - CTC (transfer stress-test): Alignment-free. Tests whether pretrained features support sequence-level decoding without position information. - CE + MFA-guided segment pooling (ablation — untested): Per-phoneme-segment mean-pool using MFA timing boundaries. - CE + learned attention pooling (ablation — untested): 3 position queries cross-attend to temporal features. Adds ~33K params.

Content-collapse diagnostics (required at evaluation): - Output entropy: H(predicted phoneme distribution) per position. Max = log2(9) ≈ 3.17 bits. If < 1.5, model collapses to a phoneme subset. - Unigram KL: KL(predicted marginal || uniform). High KL = model favors certain phonemes regardless of input. - Stereotypy index: Fraction of unique predicted 3-phoneme sequences out of total predictions. If < 10% of the 729 possible sequences appear, the model outputs stereotyped patterns. These catch failure modes that loss curves miss — the LOPO CE diagnostic showed 100% correct-length sequences but only ~21% position accuracy with concentrated phoneme distributions.

5. Architecture

Preprocessing

Neural data (Stages 2-3): 

Raw .fif (200Hz, z-scored HGA) -> anti-alias LPF -> subsample -> dead electrode mask

Anti-aliasing and temporal resolution: - Sampling rate: 20Hz (default, 30 frames/1.5s trial) or 40Hz (ablation, 60 frames). Prior evidence favors 20Hz: CE ablation on S14 showed 20Hz ≈ 40Hz (PER 0.700 at both). 40Hz alive only for masked pretraining and micro-event variants. Burden of evidence on 40Hz. - Anti-alias cutoff: Zero-phase Butterworth LPF at 0.8× Nyquist (8Hz for 20Hz, 16Hz for 40Hz). Derived from target rate, not independently chosen. - No threshold clamping on model input: z-score outliers preserved. - Prerequisite: Compute temporal PSD of real HGA before finalizing rate.

Synthetic data (Stage 1): same pipeline after generation (z-normalize, noise, dead electrodes, flips/180°, correlated noise, anti-alias LPF, subsample).

Shared Design Choices

Positional encoding — two separate concerns:

  1. Local geometry encoding (default): Each patch center in physical millimeters: (row × pitch_mm, col × pitch_mm) + dead_fraction. Linear(3 → d). Known pitches: 128ch = 1.33mm, 256ch 12×22 = 1.72mm. Default fallback for unresolved 8-row 256ch layouts (8×32, 8×34): use pitch_mm = 1.33 as an explicit surrogate derived from the known 8-row micro-ECoG family, and flag those runs as surrogate-pitch in results tables until hardware specs confirm the exact value. Normalized (x,y) in [0,1] rejected — discards physical pitch.

  2. Cross-patient anatomical encoding (future ablation): Local mm coordinates do NOT encode where the array sits on cortex. MNI offset as second PE input is a future ablation.

Coordinate-augmentation consistency (required): Flips/rotations applied to grid images must transform PE coordinate inputs identically.

Patch tokenization — three options (Priority Ablation C): - 1x1 patches (pilot default for Family B and conv-token GRU): 128-272 tokens. Preserves per-electrode resolution. - 2x2 patches (compute control): 32-68 tokens. Tests whether 2x downsampling costs spatial resolution. - Conv stem: Conv2d(1, C, k=3, pad=1) stack as spatial front-end, then flatten to tokens.

Family A: Conv-GRU (~71K pretraining) — Conservative Control

Replicates the existing supervised pipeline’s inductive biases, including its known spatial-collapse bottleneck. Spatial structure is pooled and flattened before temporal modeling. Prior training log: upstream compression (Conv2d → pool → flatten → 64-dim) is the primary bottleneck, not GRU capacity. This family deliberately preserves that bottleneck as a conservative control.

Encoder: Per-patient Conv2d(1, 8, k=3, pad=1) → ReLU → AdaptiveAvgPool2d(4, 8) → flatten → d_flat=256 → LayerNorm → Conv1d(256, d, k=1, s=1) pointwise temporal projection. Temporal downsampling handled in preprocessing, not encoder.

Predictor: BiGRU(d, d/2, 2 layers). No spatial tokens — no PE. Spatial relationships implicit in Conv2d kernel positions.

Decoder: Linear(d, 256) → reshape → upsample to (H, W).

Known limitation — bottleneck-pretext mismatch: Observation-space prediction through the 64-dim bottleneck rewards coarse reconstruction. Deliberate for the conservative control: if it still helps, pretraining is robust to lossy spatial encoding. JEPA would avoid this mismatch (deferred).

Stage 3 readout: GRU hidden states → global temporal mean pool → 27-way CE head.

Conv-Token GRU (~40K pretraining) — Bridge Architecture

Bridges Family A and Family B by preserving spatial tokens (like B) while using a GRU temporal model (like A). This isolates the spatial-token-preservation hypothesis from the GRU→attention temporal-model change — the core architecture confound.

Encoder: Per-patient Conv2d(1, 8, k=3, pad=1) → ReLU. Unlike Family A, no spatial pooling — each electrode position outputs an 8-dim feature vector. Per-position Linear(8, d) projects to d-dim tokens. Local geometry PE Linear(3 → d) added per position (row_mm, col_mm, dead_frac).

Predictor: Shared BiGRU(d, d/2, 2L) applied independently to each spatial position’s time series. Weights shared across all positions (analogous to weight-sharing in Conv2d). Each position evolves independently — no cross-position temporal interaction. This is deliberate: the only cross-position information comes from Conv2d’s 3×3 kernel and from spatial pooling at readout. If this architecture matches or beats Family A, spatial-token preservation is valuable even without cross-position attention.

Decoder: Shared Linear(d, 1) per position per frame → reconstruct per-electrode values.

Stage 3 readout: Spatial pooling (mean-pool + top-k, co-default) across positions → temporal mean-pool → 27-way CE head.

Compute note: Processing N_pos independent GRU sequences per trial (128 positions × 30 frames at batch_size=8 = 1024 sequences) is within GRU’s efficient regime. For 12×22 grids (264 positions), effective batch = 2112 — still tractable.

Comparison axes (clean deconfounding): - Family A → Conv-Token GRU: spatial collapse removed, PE added, per-position GRU. Isolates spatial-token preservation. - Conv-Token GRU → Family B: GRU → transformer, cross-position attention added, ~23× capacity. Isolates temporal model class.

Family B: ViT + Factorized Transformer (~930K pretraining)

Higher representation capacity. Requires freezing at Stage 3.

Encoder (spatial frame → spatial tokens): Per-patch tokens, spatial structure preserved.

  • d=128, 4 layers, 4 heads, head_dim=32, GELU FFN with 2x expansion
  • Patch embedding: Linear(patch_dim, 128). Shared in Stage 1, per-patient in Stages 2-3. Known weakness: Linear(1,128) at 1x1 patches is per-electrode gain+bias only — cannot model rotation, local mixing, or spatial blur. Per-patient conv stem (Conv2d(1, 128, k=3, pad=1), ~1.2K params/patient) is the paired ablation.
  • Total encoder: ~528K

Predictor (factorized space-time transformer): TimeSformer-style divided space-time attention. Each layer: temporal self-attention → spatial self-attention → FFN. d=128, 2 layers. - Total predictor: ~396K

Temporal attention is bidirectional at all stages. No pretraining→fine-tuning mask mismatch.

Decoder: Shared Linear(128, patch_dim). Discarded after pretraining.

Stage 3 readout: Spatial pooling → temporal pooling → CE head.

Spatial pooling — mean-pool and top-k as co-defaults (first round): - Mean-pool: Simple average across spatial tokens. May wash out focal activation on <2mm grids. - Top-k pool (co-default): Average only the k spatial tokens with highest L2 norm per timestep (k=16 default). Parameter-free. Selects the most active electrodes — directly addresses the concern that spatial-token-preserving architectures pay transformer cost only to average away the spatial signal. - Learned attention pooling (Tier 1 ablation): Learned query q (128-dim) cross-attends to spatial tokens. Adds ~33K params — deferred past first round.

Both mean-pool and top-k are run in the first round. If top-k >> mean-pool, promote to sole default.

Capacity Justification

The three families span ~40K to ~930K pretraining parameters. The ~330K GRU that showed 55x train/val gap was trained end-to-end from scratch — all parameters learned from ~120 trials. The pretrained models train only ~2.3-5.3K at Stage 3 (frozen features). If pretraining works, this is analogous to a pretrained vision backbone with a linear probe. If it fails, the conv-GRU (~71K) can attempt full fine-tuning as a fallback. The freeze-level ablation (Tier 1) is make-or-break.

Conv-GRU runs first (pilot). Conv-token GRU runs alongside (Step 3b). ViT only after signal confirmed. If conv-GRU matches ViT, the extra complexity was unnecessary — a valid, publishable finding.

6. Comparison Methods

Seven methods in the initial round. Conv-GRU first (pilot family); expand to ViT after signal confirmed.

Step-matched protocol: Methods A, B, C, F, and K have the same total optimizer step count S_total: - Method A: S_total/2 synthetic (Stage 1) + S_total/2 neural (Stage 2) - Method B: S_total neural (Stage 2 only, random init) - Method C: S_total/2 smooth AR + S_total/2 neural - Method F: S_total/2 random-scaffold synthetic + S_total/2 neural - Method K: S_total/2 destroyed-dynamics synthetic + S_total/2 neural - Residual confound: Step-matched ≠ information-matched. A sees unlimited synthetic diversity; B reuses ~1500 trials. Method C and K partially control for this (also unlimited). Method J (B-extended) tests whether more neural-only training closes the gap.

A. Structured Synthetic + Neural (hypothesis)

Stage 1: Masked span prediction on a mixed structured synthetic pool {switching LDS, wave, Gray-Scott, FHN, NCA, hotspot trajectory} (S_total/2 steps) → Stage 2: Neural (S_total/2 steps) → Stage 3: head. The switching LDS is part of the first-round default pool, not a deferred add-on.

B. Neural Only (step-matched)

Stage 2: Neural masked span prediction, random init (S_total steps) → Stage 3: head

C. Smooth AR + Neural (synthetic control)

Stage 1: Smooth Gaussian AR (S_total/2 steps) → Stage 2: Neural (S_total/2 steps) → Stage 3: head. Isolates whether structured dynamics matter or any smooth synthetic movie helps.

D. Supervised Baseline

No pretraining. Encoder + predictor + head trained end-to-end with CE.

E. Random-Init Frozen (reservoir baseline)

Freeze randomly-initialized encoder+predictor. Train only CE head.

F. Random-Scaffold Synthetic + Neural (scaffold control)

Stage 1: Same generators as A, but random phase count (2-5) at random timepoints (S_total/2 steps) → Stage 2: Neural (S_total/2 steps) → Stage 3: head. If A ≈ F, the 3-event scaffold doesn’t matter. First round.

K. Destroyed-Dynamics Synthetic + Neural (statistics control)

Stage 1: Same synthetic data as A (same generators, same scaffold) but with temporal dynamics destroyed — phase-randomized or time-shuffled within scaffold (S_total/2 steps) → Stage 2: Neural (S_total/2 steps) → Stage 3: head. Matches A’s spatial statistics, amplitude distribution, PSD, and event sparsity while destroying temporal structure. If A > K, dynamics are load-bearing. If A ≈ K, spatial statistics drive the benefit. First round, required for Claim 3.

Pilot-first sequencing: Conv-GRU first (simpler, faster, ~13x smaller than ViT).

7. Evaluation Framework

Primary Metrics

  • PER: via CE per-position argmax (primary) or CTC greedy decode
  • Balanced accuracy: 9-way, per position
  • Grouped-by-token CV per patient (see Stage 3)

Evaluation Mode

  • Inductive is the primary result: For target patient t, Stage 2 uses only patients in PS \\ {d, t} as defined in the Stage 2 pool table
  • Transductive is secondary: For target patient t, Stage 2 uses PS \\ {d} and includes t‘s unlabeled trials, but keeps the same synthetic calibration as the corresponding inductive fold

Required Diagnostics

  1. Interpolation baseline: Trained model must beat substantially
  2. Patient-ID linear probe: After each stage. Sharp patient-ID rise WITHOUT phoneme-ID improvement is concerning
  3. Phoneme-ID linear probe: After each stage. Tracks where task-relevant structure emerges
  4. Per-patient loss monitoring: Variance and dominance ratios during Stage 2
  5. Transfer-proxy accuracy curve: Plot vs Stage 2 step count
  6. Content-collapse diagnostic (required at Stage 3): Output entropy per position (max 3.17 bits; alarm if < 1.5), unigram KL from uniform, stereotypy index (unique 3-phoneme sequences / total predictions; alarm if < 10% of 729 possible). These catch failure modes loss curves miss — LOPO CE showed 100% correct-length sequences but only ~21% position accuracy with concentrated phoneme distributions
  7. JEPA promotion trigger (monitored continuously): (a) event-frame pretext gains don’t survive execution-only decoding, (b) patient-ID rises >2σ without phoneme-ID, (c) pretext MSE plateaus >5K steps without transfer-proxy improvement. Any one promotes synthetic JEPA / latent target prediction to the next follow-up run

Reporting

Patient-level aggregation: For each method × patient, compute mean PER across CV folds. This patient-level PER is the atomic unit. Wilcoxon signed-rank operates on paired patient-level PERs (A vs D for patient i, across N-1 headline patients). Do NOT aggregate fold-level results across patients.

Population statistics: - Per-patient table (all patients, all methods, all readout windows) - Population mean ± std of patient-level PER - Paired Wilcoxon signed-rank test (primary: A vs D, A vs B; secondary: A vs C, A vs F, A vs K) - Bootstrap 95% CIs on population mean PER difference (10K resamples) - Cohen’s d (effect size) - Holm-Bonferroni correction for multiple comparisons

Surrogate null control (required for headline result): For A vs D, permute phoneme labels within each patient (preserving patient/fold structure), retrain Stage 3 head, compute PER. 100× per patient. If real improvement falls within null 95th percentile, pretraining gain is not label-informative.

8. Ablations (Prioritized)

Priority ablations (run with initial round, conv-GRU pilot)

A. Pretext Objective — “Is the pretext task right?” - Masked temporal spans (default: 40-60% masking) vs causal multi-step prediction at k={1,3,5} frames vs masked spans with varied span lengths

B. Temporal Resolution — “Is the input right?” - 20Hz (default, 30 frames/trial) vs 40Hz (60 frames/trial). Lock early.

C. Patch Size / Spatial Resolution — “Is the tokenization right?” - 1x1 (default) vs 2x2 vs conv stem. Micro-ECoG at <2mm pitch is where single-electrode resolution matters.

D. Masking Ratio — “Is the pretext hard enough?” - 30%, 50%, 70% of frames masked (default 40-60%). Interacts with span count and temporal resolution. Smooth 20Hz HGA may require >50% masking to prevent interpolation solutions. At 40Hz, 50% masking provides more boundary context than at 20Hz.

Tier 1 (run after priority ablations show movement)

Gate: Run Tier 1 only if Method A > Method E. If A ≈ E, debug pretext first.

E. Synthetic Generator Family (ladder): Single-family runs (AR, Switching LDS, Wave, Gray-Scott, FHN, NCA, Hotspot, Mixed).

F. Freeze Level: Head+embed+LN (default) vs head only vs full fine-tune.

G. Architecture Family: Conv-GRU vs conv-token GRU vs ViT. Conv-token GRU and ViT spot-checks are first-round (Steps 3b, 4b). Full expansion is Tier 1.

H. Spatial Pooling (ViT and conv-token GRU): Mean-pool vs top-k vs learned attention. Mean-pool and top-k are first-round co-defaults; learned attention is Tier 1.

I. Source Weighting in Stage 2: Uniform vs similarity-weighted vs loss-balanced.

Tier 2 (run if Tier 1 shows promise)

  • Latent dimension: d in {64, 128, 256}
  • Stage 2 data volume: 2, 4, 8, all patients
  • Stage 3 loss: CE primary → also test CTC, CE+MFA, CE+learned attention
  • Lexical patients in Stage 2
  • SIGReg: lambda in {0, 0.01, 0.1}
  • Predictor depth: 2 vs 4 layers
  • Coordinate PE: local mm vs MRoPE vs normalized (x,y) vs MNI-augmented
  • Regime switch structure variants
  • 90-degree rotation augmentation
  • Causal vs bidirectional temporal attention at Stage 3

9. Known Risks

  1. Synthetic-to-neural domain gap: Bridged by nuisance augmentation, regime-switching, coarse calibration. Method C isolates whether structure matters; Method K isolates whether dynamics matter. Matching statistics ground the transfer expectation.
  2. Trivial interpolation solution: Addressed by 40-60% masking (not 10-17%), interpolation baseline, and event-frame diagnostic.
  3. Trial-calendar leakage: Response-locked trials have stereotyped phase structure. Active mitigations: full-epoch off-center crops, time-reversal augmentation. Diagnostics: phase-template/patient-template/cropped-window baselines, per-phase loss tracking. Stage 2 is honestly framed as response-locked trial adaptation. If gains disappear at execution-only decoding, the paper’s claims narrow accordingly.
  4. Content collapse: Prior LOPO showed CTC blank collapse (95%) and CE stereotyped outputs (~21% accuracy). Content-collapse diagnostics (entropy, unigram-KL, stereotypy) track this at Stage 3.
  5. Statistical power: N=8-12 with noisy estimates. Bootstrap CIs, effect sizes, per-patient plots, label-permutation surrogate null.
  6. FHN parameter instability: Map interesting regime before generating. Coarse calibration catches degenerate instantiations.
  7. ViT overfitting despite freezing: ~5.3K trainable at Stage 3 is small but not zero. Conv-GRU is the fallback.

9b. Precautionary Lessons from Prior Experiments

  • Data scarcity is the wall: H=128 showed 55x train/val gap. Freeze-level ablation is make-or-break.
  • LOPO failed near chance (0.846): Stage 2 pooled training faces the same risk — hence the transfer-proxy gate.
  • CE beats CTC per-patient (0.700 vs 0.778): The 0.700 used full-window global mean-pool. Neither motor-focused crop nor grouped-by-token CV match this — 0.700 is evidence of CE > CTC, not a direct target.
  • Acoustic regression pivot failed: 500ms-shifted controls competitive. Trial structure, not alignment, was learned. Stage 2 anti-leakage directly addresses this.
  • Auxiliary targets need negative controls: Method E (random-init) is the negative control here.
  • MPS breaks with mixed grid sizes: Test ALL grid sizes early.
  • Per-patient adaptation is the default assumption (Singh 2025, LOPO pilot failure).

13. Open Questions

  1. Masked spans vs causal multi-step: Default is masked spans (bidirectional); causal multi-step is an ablation. If causal >> masked, the forward-prediction inductive bias matters more than bidirectional context.
  2. FHN parameter ranges: Map interesting regime from phase diagrams.
  3. Multi-variable observation: FHN has 2 variables per cell. Expose both (H, W, 2) or only observable (H, W, 1)?
  4. Prediction target space: Predict raw frame patches or encoder-output patch tokens (with stop-gradient)?
  5. Predictor depth: 2 layers may be insufficient. Tier 2 ablation (2 vs 4 layers).
  6. Array pitch for 8×32 and 8×34 grids: Confirm exact physical pitch values from hardware docs or Zac. Current default uses a transparent 1.33 mm surrogate for unresolved 8-row 256ch layouts and flags those runs as surrogate-pitch.
  7. Dev patient selection: Criteria: median trial count, signal quality, representative grid. Candidate: S26 (151 trials, 8×16, 74 sig channels). Confirm before experiments.
  8. Motor-focused crop boundaries: Default [-0.15, +0.6s] is approximate. Exact boundaries from per-patient MFA timing. Patient-specific vs fixed crop is open.
